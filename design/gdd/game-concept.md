# Game Concept: Timokha Escape (рабочее название)

*Created: 2026-06-24*
*Status: Draft*

> **Правовая заметка.** Игра — самостоятельный **оммаж жанру** «Побег от …»
> (мем-игры), а не копия конкретного продукта. Имена персонажей, ассеты,
> код и тексты — оригинальные/плейсхолдерные. Бренды и чужие материалы не используются.

---

## Elevator Pitch

> Казуальный квест-комедия с погоней: шкодливый деревенский шалун **постоянно
> носится за тобой** по локации, а ты на бегу выполняешь череду абсурдных поручений
> (мини-игр), уворачиваясь от него, чтобы в итоге удрать. Это «WarioWare встречает
> деревенский мем под вечной погоней».
>
> *10-секундный тест:* бегаешь от шалуна → добегаешь до объекта → быстрая мини-игра →
> смешная реакция → открывается следующее поручение → шалун снова на хвосте → в финале главы вырываешься на свободу.

## Core Fantasy

Игрок — комичный аутсайдер, попавший в нелепую деревенскую передрягу, **по пятам за
которым вечно носится шкодливый шалун**. Его «суперсила» — не боевые навыки, а
смекалка и расторопность: на бегу, уворачиваясь от преследователя, он превращает
абсурдные хлопоты (налепить пирожков, заварить чай, наколоть дров, накормить шалуна
огурцом) в пошаговый план побега. Фантазия — **под непрерывной комичной погоней
перехитрить дурашливого антагониста и удрать, смеясь**. Нигде больше нет именно этого
добродушно-мемного деревенского абсурда «успей сделать дело, пока тебя догоняют».

---

## Unique Hook

Как игры жанра «Побег от …», **И ТАКЖЕ** — антагонист **присутствует постоянно и
непрерывно гонится** за игроком, пока тот выполняет цепочку трёхсекундных мем-мини-игр.
Хук — не в страхе (как у Granny-клонов), а в **комедийном напряжении погони**:
«успей сделать дело, пока тебя догоняет шалун» + панчлайн-реакция на каждый успех = «ещё разок».

## Player Experience Analysis (MDA Framework)

### Target Aesthetics (What the player FEELS)

| Aesthetic | Priority | How We Deliver It |
| ---- | ---- | ---- |
| **Sensation** (sensory pleasure) | 3 | Сочный «джус»: тряска, частицы, чавкающие/смешные звуки на каждый тап и «осаливание» |
| **Fantasy** (make-believe) | 4 | Деревенский мем-сеттинг, нелепый антагонист-преследователь, роль пройдохи-беглеца |
| **Narrative** (drama, story arc) | 1 | Комедийная арка побега, мем-реплики между задачами (панчлайны, не катсцены) |
| **Challenge** (mastery) | 2 | Комичное напряжение погони: уворачиваться от шалуна, успевая делать дела; + мастерство мини-игр (звёзды) |
| **Fellowship** (social) | N/A | Одиночная игра |
| **Discovery** (exploration) | 5 | Маленькие пасхалки и реакции объектов в локациях |
| **Expression** (creativity) | 6 | Косметика персонажа (поздняя фича) |
| **Submission** (relaxation) | 3 | Низкострессовый луп: провал не наказывает, мгновенный ретрай |

### Key Dynamics (эмерджентное поведение)
- Игроки **перепроходят** мини-игры ради «трёх звёзд» и идеального времени побега.
- Игроки **исследуют** локацию тапами в поисках смешных реакций до того, как взяться за задачу.
- Игроки **делятся** мем-моментами (скриншоты/клипы) — вирусный потенциал жанра.

### Core Mechanics (системы, которые строим)
1. **Свободное передвижение по локации** — вирт. джойстик/перетаскивание; игрок бегает по сцене, уворачиваясь от преследователя, и подбегает к «горячим точкам» для взаимодействия (тап).
2. **Постоянный преследователь (Pursuer AI)** — шалун **всё время** гонится за игроком по локации (простое преследование/steering, прощающая скорость, телеграфированные рывки). Догнал → беззлобно-комичный сетбэк (роняешь прогресс текущей задачи, пара секунд «оглушения», отскок).
3. **Движок мини-игр** — единый интерфейс для коротких задач (свайп/тап/удержание/ритм), с оценкой (звёзды). У «горячей точки» погоня **паузится/замедляется** (безопасное окно), но надо успеть добежать и доделать; прерывание — если не успел подготовиться.
4. **Квест-цепочка** — менеджер задач: какие поручения открыты, что разблокирует следующее, что открывает финальный выход.
5. **Финальный рывок-побег** — кульминация главы: добежать до выхода/яхты, оторвавшись от шалуна.
6. **Реакции и «джус»** — система обратной связи (анимации, частицы, звук, мем-реплики), в т.ч. на «осаливание» шалуном.

## Player Motivation Profile

### Primary Psychological Needs Served

| Need | How This Game Satisfies It | Strength |
| ---- | ---- | ---- |
| **Autonomy** | Свобода тапать что угодно в локации, выбирать порядок части поручений | Supporting |
| **Competence** | Быстрое освоение мини-игр, звёзды и рекорды как мерило роста | Core |
| **Relatedness** | Привязанность к комичным персонажам и их мем-репликам | Supporting |

### Player Type Appeal (Bartle Taxonomy)

- [x] **Achievers** — собирают звёзды, проходят главы, бьют рекорд времени побега
- [x] **Explorers** — тыкают в объекты ради смешных реакций и пасхалок
- [x] **Socializers** — делятся мем-моментами (скриншоты/клипы), но без онлайн-механик
- [ ] **Killers/Competitors** — не целевые (соревнование только с собственным рекордом)

### Flow State Design
- **Onboarding curve**: первая мини-игра учит одним жестом, без текста — «покажи, не рассказывай».
- **Difficulty scaling**: мини-игры чуть ускоряются/усложняются к финалу главы; побег — пик напряжения.
- **Feedback clarity**: мгновенные звёзды, цифры, анимация — игрок всегда видит результат.
- **Recovery from failure**: провал = смешная реакция + мгновенный ретрай, без штрафа и экранов «Game Over».

## Core Loop

### Moment-to-Moment (30 секунд)
Бегать от шалуна по локации → улучить момент и добежать до подсвеченного объекта →
тапнуть → быстрая мини-игра (свайпы/тапы/удержание), держа дистанцию → смешная реакция
и награда (звёзды). Постоянная лёгкая угроза погони + «джус» делают момент бодрым и смешным.

### Short-Term (5-15 минут)
Выполнить цепочку из 2-3 поручений в текущей локации. Каждое выполненное поручение
открывает следующее или предмет, нужный для побега. «Ещё одну задачу» — за счёт юмора и быстрого результата.

### Session-Level (30-120 минут → здесь короче)
Пройти главу-локацию: например, **кухня → двор → причал**, всё время уворачиваясь от
шалуна. Кульминация главы — **финальный рывок-побег** (оторваться и добежать до выхода/яхты).
Естественная точка остановки = конец главы. Причина вернуться = следующая глава и охота за тремя звёздами.

### Long-Term Progression
Новые главы/локации, косметика персонажа, «звёзды» за идеальные мини-игры,
рекорд времени побега. Игра «пройдена», когда закрыты все главы на три звезды.

### Retention Hooks
- **Curiosity**: что за следующая нелепая задача и локация?
- **Investment**: накопленные звёзды и прогресс по главам.
- **Social**: мем-моменты, которыми хочется поделиться.
- **Mastery**: добить мини-игры на три звезды, побить личный рекорд побега.

## Game Pillars

### Pillar 1: Юмор превыше всего
Каждое действие должно вызывать улыбку; абсурд важнее реализма и «правильности».

*Design test*: если выбор между «логичной» механикой и «смешной» — выбираем смешную.

### Pillar 2: Понятно с первого касания
Любая мини-игра осваивается за ~3 секунды одним жестом, без обучающего текста.

*Design test*: если мини-игре нужен туториал — упрощаем механику, а не добавляем подсказку.

### Pillar 3: Цепочка поручений ведёт к побегу
Прогресс — это короткие задачи, каждая приближает побег; никакого «филлера».

*Design test*: при выборе контента добавляем короткую мини-игру-поручение, а не длинный уровень.

### Pillar 4: Карманные сессии
Играется урывками по 2-5 минут; всегда есть быстрая точка входа и выхода.

*Design test*: если обязательная сессия удлиняется свыше ~5 минут — режем или делаем опциональной.

### Anti-Pillars (What This Game Is NOT)

- **НЕ хоррор**: антагонист смешной, не страшный — пугающий тон убил бы пиллар «Юмор превыше всего».
- **НЕ хардкорные тайминги/платформинг**: точные реакционные испытания отпугнут казуальную аудиторию (нарушение «Понятно с первого касания»).
- **НЕ длинные катсцены и визновелла**: юмор подаётся короткими мем-репликами, а не пятиминутными роликами (нарушение «Карманные сессии»).
- **НЕ агрессивная монетизация / pay-to-win**: ломает доброжелательный тон и доверие аудитории.

## Risks and Open Questions

### Design Risks
- Мини-игры могут приедаться, если механики однотипны — нужен разнообразный набор жестов.
- Юмор субъективен; «несмешные» реплики ослабят главный пиллар.
- **Баланс погони**: слишком быстрый/«липкий» шалун = фрустрация и конфликт с анти-пилларом *НЕ хардкорные тайминги*; слишком медленный = угроза не ощущается. Нужна прощающая, телеграфированная погоня (проверить прототипом).
- **Управление под погоней**: выполнять мини-игру, пока преследователь рядом, не должно требовать «третьей руки» — мини-игра либо ставит погоню на паузу/замедление у «горячей точки», либо даёт безопасное окно.

### Technical Risks
- «Движок мини-игр» должен быть достаточно общим, чтобы дешево добавлять новые — риск переусложнить абстракцию.
- Тач-зоны и «джус» на слабых Android-устройствах в рамках 60 FPS.

### Market Risks
- Жанр насыщен дешёвыми клонами — выделяемся качеством «джуса», UX и юмора.
- Низкая платёжеспособность аудитории при premium-модели (отсюда — осторожная монетизация позже).

### Scope Risks
- Соблазн «ещё одной мини-игры» — контролируется пилларом «Цепочка поручений» и анти-пилларами.

### Open Questions
- Сколько мини-игр на главу даёт лучший ритм под погоней? → проверить прототипом (`/prototype`).
- *(Решено: у «горячей точки» погоня паузится/замедляется — безопасное окно для мини-игры; точную степень (пауза vs замедление) калибруем прототипом.)*
- Какая модель «осаливания» приятнее: сброс прогресса задачи vs потеря звезды vs просто потеря времени? → проверить прототипом.
- *(Решено по запросу пользователя: антагонист присутствует ПОСТОЯННО и гонится всю главу — как в референсе, а не только в финале.)*

## MVP Definition

**Core hypothesis**: «Игрокам смешно и приятно делать цепочку коротких мини-игр,
**убегая от постоянно преследующего шалуна**, в сессии 2-5 минут».

**Required for MVP**:
1. Одна локация (кухня) со **свободным передвижением** и 3 «горячими точками».
2. **Постоянный преследователь (шалун)**: гонится за игроком по локации, прощающая скорость; «осалил» → комичный сетбэк (сброс прогресса текущей задачи + короткое оглушение + отскок).
3. Движок мини-игр + 3 разные мини-игры (напр.: лепка пирожков — ритм-тапы; заварка чая — удержание; колка дров — свайпы) с оценкой в звёздах.
4. Квест-цепочка, открывающая задачи по порядку, и финальный рывок-побег к выходу.
5. Базовый «джус»: анимация/частицы/звук + 1-2 мем-реплики на задачу и на «осаливание».

**Explicitly NOT in MVP** (отложить):
- Несколько глав/локаций, косметика персонажа, постоянный таймер угрозы, метапрогрессия, монетизация.

### Scope Tiers (if budget/time shrinks)

| Tier | Content | Features | Timeline |
| ---- | ---- | ---- | ---- |
| **MVP** | 1 локация, 3 мини-игры, 1 побег | Кор-луп + звёзды | ~2-3 недели |
| **Vertical Slice** | 1 полная глава с полировкой | Кор-луп + прогрессия + UX | ~3-4 недели |
| **Alpha** | Все главы (плейсхолдер-арт) | Все системы, черновой контент | ~6-8 недель |
| **Full Vision** | 4-6 глав, 15-20 мини-игр, косметика | Полный контент и полировка | ~3-4 месяца (соло) |

## Visual Identity Anchor

*(Lean-режим: гейт art-director пропущен; ниже — стартовый якорь, финализируется в `/art-bible`.)*

- **Направление:** «Деревенский мем-мультик» — тёплый, дурашливый, нарочито простой.
- **Одно правило:** *всё должно быть читаемо силуэтом и слегка «пружинить»* — объекты
  реагируют на тап скуошем/растяжением, чтобы каждое действие ощущалось мультяшно-живым.
- **Принципы:**
  1. *Крупные читаемые формы* — UI и объекты понятны на маленьком экране в движении. *Тест:* если объект не узнать по силуэту — упрощаем форму.
  2. *Тёплая палитра деревни* — земляные/тёплые тона + 1 яркий акцент на интерактивных точках. *Тест:* если игрок не видит, что тапать — усиливаем акцентный цвет.
  3. *Мультяшный «джус»* — преувеличенные реакции важнее реализма. *Тест:* спор реализм/выразительность → выбираем выразительность.
