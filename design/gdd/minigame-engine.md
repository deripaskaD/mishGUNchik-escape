# GDD — Minigame Engine (Движок мини-игр)

*Status: Approved (solo) · 2026-06-24 · System #6*

## 1. Overview
Единый расширяемый интерфейс коротких мини-игр-поручений. Добавление новой мини-игры =
новый класс-наследник `MinigameBase` + запись в данных главы. Оценка — звёзды (1–3).

## 2. Player Fantasy
«Быстрая дурашливая задачка, которую я осваиваю за пару секунд и хочу сделать на 3 звезды».

## 3. Detailed Rules
- Мини-игра активируется автоматически, когда игрок входит в зону текущего поручения.
- Ввод маршрутизируется в активную мини-игру: `input_tap()`, `input_press(down)`, `input_dir(±1)`.
- Прогресс 0..1; по достижении 1 — `is_done() == true`, считаются звёзды.
- Три стартовые мини-игры: **pies** (мэш-тапы), **tea** (удержание до целевой зоны и отпускание), **wood** (чередование Л/П).

## 4. Formulas
- pies: `progress += 1/target_taps` за тап.
- tea: меч-метр `fill += fill_speed*delta` при удержании; успех при отпускании в полосе `|fill-target|<=band`; иначе `mistakes++`, меч сбрасывается.
- wood: `progress += 1/target_chops` за корректное чередование; неверная сторона → `mistakes++`.
- Звёзды: `3` если `time<=par и mistakes==0`; `2` если `time<=par*1.6 и mistakes<=2`; иначе `1`.

## 5. Edge Cases
- Поимка шалуном → `reset()` прогресса (звёзды не начисляются повторно).
- Переполнение tea-метра (>1) → промах, не «застревание».
- Доходимость: при `target_taps`/`target_chops` достаточно малых, чтобы успеть под погоней.

## 6. Dependencies
Input & Touch Controls, App & Game State. Используется Quest Chain (через сигнал `minigame_finished`).

## 7. Tuning Knobs
`target_taps`, `target_chops`, `target`, `band`, `fill_speed`, `par_time` — в данных задачи (`GameData`).

## 8. Acceptance Criteria
- [ ] Три мини-игры играются и завершаются.
- [ ] Звёзды считаются по формуле; детерминированно (тест).
- [ ] Новая мини-игра добавляется без правки движка (только наследник + данные).
- [ ] Поимка корректно сбрасывает прогресс.
