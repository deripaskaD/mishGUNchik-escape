# GDD — Quest Chain / Task Manager + Escape

*Status: Approved (solo) · 2026-06-24 · Systems #8 + #9*

## 1. Overview
Менеджер цепочки поручений главы: какая задача активна, что разблокирует следующую,
когда открывается финальный выход (побег). State-machine над списком задач главы.

## 2. Player Fantasy
«Каждое выполненное дело приближает побег — видно, что я прогрессирую».

## 3. Detailed Rules
- Задачи выполняются **по порядку**; активна только текущая (её зона/хотспот видим).
- Выполнение мини-игры текущей задачи → запись звёзд, переход к следующей.
- Когда все задачи выполнены → открывается **ВЫХОД**; шалун на полной скорости (без замедления).
- Достижение выхода → глава завершена; итоговые звёзды главы = округлённое среднее по задачам.

## 4. Formulas
- `index` растёт на 1 за выполненную задачу; `all_done := index >= tasks.size()`.
- `chapter_stars = round(mean(task_stars))`, минимум 1.

## 5. Edge Cases
- Поимка во время задачи не сбрасывает уже выполненные задачи (только прогресс текущей мини-игры).
- Глава без задач (защита) → сразу фаза побега.

## 6. Dependencies
Minigame Engine, Hotspot/Interaction, Player Movement, Pursuer AI. Пишет в GS (прогрессия/сохранение).

## 7. Tuning Knobs
Состав и порядок задач, позиции хотспотов, позиция выхода — в `GameData.CHAPTERS`.

## 8. Acceptance Criteria
- [ ] Задачи идут по порядку; активна только текущая.
- [ ] После всех задач появляется выход; достижение = завершение главы.
- [ ] Итоговые звёзды считаются и сохраняются (лучший результат).
- [ ] Тест state-machine проходит.
