# ADR Log — Timokha Escape

*Solo-режим, 2026-06-24. Каждый ADR: контекст → решение → последствия.*

## ADR-0001 — Логика отделена от визуала
**Контекст:** мобильная игра, визуал плейсхолдерный и заменяемый.
**Решение:** `GameData`/`GS` (данные/состояние) не знают о нодах отрисовки; `World`/UI читают состояние.
**Последствия:** + тестируемая логика, drop-in арт; − немного бойлерплейта на чтение состояния.

## ADR-0002 — Immediate-mode `_draw` для геймплея
**Контекст:** нет арт-ассетов, нужна играбельность без графики.
**Решение:** геймплей рисуется фигурами через `_draw()`; финальный арт — drop-in `art/<key>.png`.
**Последствия:** + ноль зависимостей от ассетов; − при переходе на спрайты следить за draw calls (ADR-0006).

## ADR-0003 — `MinigameBase` как точка расширения
**Контекст:** много коротких мини-игр, нужно дёшево добавлять.
**Решение:** базовый RefCounted-класс с единым интерфейсом ввода/прогресса/звёзд; конкретные — наследники.
**Последствия:** + новая мини-игра = класс + данные; − общий интерфейс должен покрывать разные виды ввода (tap/hold/dir).

## ADR-0004 — Данные-первичны (`GameData.CHAPTERS`)
**Контекст:** контент (главы, задачи, тюнинг) меняется чаще кода.
**Решение:** главы/задачи/тюнинг — в данных автозагрузки, не в сценах.
**Последствия:** + контент-итерация без правки кода; − структура словарей должна быть стабильной.

## ADR-0005 — Сохранение в `user://save.json`
**Контекст:** нужна прогрессия (открытые главы, лучшие звёзды) на iOS/Android.
**Решение:** JSON в `user://` (best_stars по chapter_id, unlocked). Простая схема + версия.
**Последствия:** + переносимо, читаемо; − не защищено от подмены (для премиум-сингла приемлемо; см. /security-audit позже).

## ADR-0006 — Рендер Mobile, тач-первичный ввод
**Контекст:** iOS+Android, ландшафт.
**Решение:** renderer=mobile; ввод — плавающий джойстик + контекстные кнопки; клавиатура для desktop-теста.
**Последствия:** + целевая платформа; − следить за тач-зонами/масштабом UI под разные экраны.

## ADR-0007 — Тесты через headless-харнесс (без GUT-аддона)
**Контекст:** аддоны не установлены, нужна детерминированная проверка логики.
**Решение:** `tests/run_tests.gd` — SceneTree-скрипт с ассертами, запуск `--headless --script`.
**Последствия:** + ноль зависимостей, работает в CI; − нет богатого репорта GUT (достаточно для MVP).
