# Control Manifest — Timokha Escape

*Плоский лист правил для реализации. Извлечён из ADR + technical-preferences.*

## DO
- Держи логику в `GameData`/`GS`/RefCounted-классах; визуал — в `World`/UI.
- Все настраиваемые числа — в `GameData` (тюнинг/главы), не хардкодь в сценах.
- Новая мини-игра = наследник `MinigameBase` + запись задачи в `GameData`.
- Типизируй переменные/возвраты где можно; сигналы — в прошедшем времени.
- Рисуй геймплей через `_draw` + `queue_redraw()` раз в кадр.
- Тач-цели крупные; поддержи и клавиатуру (desktop-тест).
- Логику покрывай тестами в `tests/run_tests.gd`.

## DON'T
- Не давай `GameData`/`GS` знать о нодах отрисовки/сценах.
- Не используй `:=` для значений из словарей (вывод типа из Variant — ошибка GDScript).
- Не делай Game Over при поимке — только комичный сетбэк.
- Не добавляй внешние аддоны без необходимости.
- Не тяни код из `prototypes/` в продакшен (прототип — throwaway).

## Per-system
- **Pursuer:** всегда телеграфируй рывок; уважай замедление у зоны; грейс после поимки.
- **Minigame:** прогресс 0..1; звёзды детерминированы; поимка = reset прогресса.
- **Quest:** строго по порядку; выход после всех задач; сохраняй лучший результат.
- **Save:** `user://save.json`, версия схемы, не падай на отсутствии/битом файле.
